# AI 智能客服 PRD

## 1. 背景

当前电商系统已经具备商品、库存、购物车、结算、订单、优惠券、支付等核心能力。为了降低用户咨询成本、提升购买转化率、增强系统智能化体验，需要新增 AI 智能客服能力。

AI 客服作为用户与电商系统之间的自然语言交互入口，负责理解用户需求，并通过已有业务服务完成查询、推荐和自动化操作。

## 2. 产品目标

AI 客服需要实现以下目标：

1. 支持用户与 AI 进行自然语言闲聊。
2. 支持查询商品、库存、订单、优惠券、购物车、结算单等信息。
3. 支持根据用户需求推荐商品列表。
4. 支持自动化操作，例如添加购物车、移出购物车、领取优惠券、创建订单、取消订单。
5. 对高风险操作进行用户确认。
6. 记录 AI 的工具调用、确认行为和执行结果，满足审计和排查需求。

## 3. 目标用户

- 普通用户：通过聊天完成咨询、查询、推荐、购物车和订单操作。
- 客服/运营人员：减少重复问题处理，查看 AI 处理记录。
- 系统管理员：配置模型、工具权限、风控策略和审计规则。

## 4. 核心功能

### 4.1 实时聊天

- 提供 WebSocket 聊天入口。
- 支持多轮上下文。
- 支持流式回复。
- 支持用户确认、取消、重新提问。
- 普通闲聊不调用业务工具，直接由模型回复。

### 4.2 查询能力

AI 客服需要支持以下查询：

- 商品列表、商品详情、商品分类、价格、销量、库存。
- 商品实时库存。
- 订单列表、订单详情、订单状态。
- 优惠券列表、优惠券详情、我的优惠券、优惠券使用记录。
- 购物车列表。
- 结算单列表、结算单详情。

### 4.3 商品推荐

用户可用自然语言描述需求，例如：

- “推荐几款适合学生党的手机”
- “有没有 100 元以内的礼物”
- “帮我找热销的数码产品”

AI 需要：

- 提取关键词、分类、价格区间、热销/新品偏好。
- 调用商品查询或推荐服务。
- 返回商品列表。
- 给出简短推荐理由。
- 推荐结果包含商品 ID、名称、价格、库存、图片、分类。

### 4.4 自动化操作

支持操作：

- 添加商品到购物车。
- 减少购物车商品数量。
- 删除购物车商品。
- 领取优惠券。
- 创建预结算。
- 创建订单。
- 取消订单。

### 4.5 操作确认策略

无需确认：

- 闲聊。
- 查询。
- 商品推荐。
- 添加购物车。
- 减少购物车商品数量。
- 领取优惠券。

必须确认：

- 删除购物车商品。
- 创建订单。
- 取消订单。
- 使用优惠券参与下单。
- 后续支付、退款、地址修改等敏感操作。

确认请求需要包含：

- 确认 ID。
- 操作名称。
- 操作摘要。
- 过期时间。
- 待执行参数摘要。

## 5. 安全要求

- 用户 ID 必须来自登录态，不允许客户端或模型传入覆盖。
- AI 只能访问当前登录用户的数据。
- 高风险操作必须通过确认 ID 校验。
- 确认请求过期后不得执行。
- 同一个确认请求不能重复执行。
- 工具调用失败时不得伪造成功回复。
- 所有写操作必须记录审计日志。

## 6. 非功能要求

- WebSocket 首包响应目标：2 秒内。
- 查询类工具调用超时：3 秒。
- 写操作工具调用超时：5 秒。
- 支持 LLM Provider 可插拔。
- 默认兼容 OpenAI-compatible API 和 DeepSeek。
- 模型不可用时降级为提示用户稍后重试。
- 支持用户级、工具级限流。
- 支持日志、指标和链路追踪。

## 7. 验收标准

- 用户可以通过 WebSocket 与 AI 连续对话。
- 用户可以查询商品、库存、订单、优惠券、购物车。
- 用户可以获得商品推荐列表。
- 用户可以通过 AI 添加购物车、领取优惠券。
- 用户取消订单或创建订单前，系统必须要求确认。
- 过期或重复确认不能执行。
- 工具调用结果与现有 RPC 返回一致。
- 审计记录可追踪每次工具调用和写操作。
